在阶段一完成之后我们谈到剩余几个问题：

1. 前端仍然是 mock 数据
   console-demo.ts 里写死了项目状态、阶段、任务、事件、Agent、产物等数据。

2. 前端和 backend 还没有真正接线
   目前 README 也明确说当前版本优先验证页面壳、布局切分、组件职责和 API 接线点。

3. Worker 执行器还没有真正成为主链路
   后端能接收 worker.progress / blocker / result，但真实 Manim Worker / HTML Worker / Reviewer 还没有被统一执行器调起来。

4. Context Packet 仍偏 metadata scaffold
   还不是读取真实论文、笔记、公式、脚本正文后拼出的 assembled prompt。

5. 产物预览仍是假数据
   讲解脚本、分镜、Manim 动画、审核报告现在只是前端展示卡片，还没有绑定 outputs。

先利用codex和claude交叉验证修补思路然后让ai自动执行，这一部分内容都是逻辑链路的链接没有什么需要指导ai的，等待他执行完毕即可。

在执行完毕之后，可以根据实际情况查看代码，一般来说会存在很多回退机制，甚至没接入api，大部分角色都是乱的，甚至调用同一个接口。根据实际情况指导ai修复这些问题，在这一阶段退回机制是完全不必要的，可以要求ai删除回退机制，把所有断点都暴露出来方便修复。

这一步和[[tts]]是可以并行处理的。
